Road Traffic Accidents Detection Based On Crash Estimation

Introduction

An autonomous system based on computer vision techniques that detects road accidents and reports them in real-time as well as allowing the monitoring of accidents using a client server architecture and an interactive GUI.

IEEE Research Paper

This paper proposes a framework to detect road accidents in real-time using generic CCTV cameras installed on roads. The framework focuses on achieving high performance on congested roads by introducing a new technique called track-compensated frame interpolation (TCFI) for tracking vehicles, aiming for higher accuracy through a new crash detection approach, and managing a large number of CCTV cameras efficiently.

System Architecture

Framework Architecture

The framework consists of 4 phases; it starts with a vehicle detection phase using YOLO architecture. The second phase is the vehicle tracking using MOSSE tracker. Then the third phase is a new approach we introduce to detect crash based on crash estimation. Finally, we can consider either what remains after the third phase is a crash or start the fourth phase, crash detection using violent flow descriptor.

Dataset

We have two types of datasets:

  1. One for training the ViF descriptor located in the "dataset" folder.
  2. The other consists of videos ranging from 20 to 120 seconds, found in the "videos" folder.

When running the client app, use the second type of dataset.

Installation

  1. Clone the repository: 
    git clone https://github.com/khaledsabry97/Argus.git
  2. Change to the project directory: 
    cd Argus
  3. Install the required dependencies: 
    pip install -r requirements.txt

Usage

To run the project, use the following commands:

  1. Run backend services: 
    python RunMaster.py
    python RunDetect.py
    python RunTracker.py
    python RunCrash.py
  2. Run client services: 
    python RunGui.py
    python RunCamera.py
  3. After the client app opens, select a video from the videos folder then hit process!

How to Configure Used Modules in Project

As the project has different settings, there are hyperparameters you need to configure to use the module you want. Go to the file called "Constants.py" in "Argus/System/Data/Constants.py"

In Detection Module
  • Work_Detect_Files: True if you want to use already saved vehicles detected in the videos provided in the project instead of using YOLO architecture.
In Tracking Module
  • Work_Tracker_Type_Mosse: True if you want to use Mosse Tracker instead of Dlib Tracker.
  • Work_Tracker_Interpolation: True if you want to use track-compensated frame interpolation (TCFI) instead of the normal tracking algorithm.
In Crash Module
  • Work_Crash_Estimation_Only: True if you want to use Crash Estimation Module Only, instead of following the Crash Estimation Module with ViF Description.
